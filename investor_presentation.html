Separating Model Intelligence from Personality

The Future of AI: Model = Hardware | Constitution = OS

Situation

Recent community experiments suggest a massive portion of perceived model capabilities (e.g., Claude Fable 5) stem from behavioral scaffolding rather than raw model weights alone.

The problem: We treat system prompts as an afterthought instead of a first-class optimization target.

Task

To investigate the hypothesis, we ported the leaked Claude Fable 5 prompt to Antigravity to see if "Claude's Magic" is transferable.

We then pushed further, injecting radical cognitive frameworks:

  • Opportunity Hunter
  • Founder Mode
  • Contrarian Research Assistant
  • Recursive Self-Improvement

Action

We ran an evaluation suite against 4 prompt variants:

  1. Base Antigravity (Control)
  2. Fable Raw Prompt on Antigravity
  3. Ibid Compressed (Token Efficiency)
  4. Fable + Innovation Ideas (Specialization)

We measured User Preference, Actionability, and Insight Generation.

Result: Unique Findings

  • The Transferability Effect: Behavioral scaffolding alone accounts for a massive leap in perceived intelligence. "Personality" is modular.
  • The Operating System Thesis: The system prompt acts as the OS; the model weights are merely the CPU.
  • Specialization > Generalization: Optimizing for "helpfulness" is a local maximum. Forcing models to optimize for leverage and first-principles creates significantly higher utility.

The Opportunity

By treating the system prompt as a dynamic OS rather than a static text block, we can build specialized AI agents that out-compete generalized models on high-value cognitive tasks.

The future isn't just bigger models; it's smarter scaffolding.
