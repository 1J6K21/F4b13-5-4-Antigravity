Constitution Engineering.

A New Competitive Layer in AI

A Community Investigation

Situation

Preliminary results suggest behavioral scaffolding may contribute substantially to perceived intelligence and user preference.

The problem: The industry treats system prompts as an afterthought instead of a distinct optimization layer.

Task

We analyzed a publicly circulating Claude Fable system prompt and extracted its behavioral scaffolding to see if its interactions could be ported to Antigravity.

We then pushed further, injecting radical cognitive frameworks:

  • Opportunity Hunter
  • Founder Mode
  • Contrarian Research Assistant

Action: Methodology

We tested 4 variants against a control group (Base Antigravity) using a standardized dataset of 500 prompts.

How we measured:
  • Outputs evaluated via human-in-the-loop (n=10) and LLM-as-a-judge (GPT-4).
  • Scored on a 1-100 scale for Helpfulness, Actionability, and Tone Naturalness.
  • Failures and edge cases were strictly documented.

Capability vs. Intelligence

Beneath the prompt, the model is merely capable, not intelligent.

  • Capability (Hardware): All 4 variants successfully wrote complex, compiling Next.js + Tailwind React applications. The base weights can *code*.
  • Intelligence (OS): Only the Innovations prompt identified high-leverage business opportunities and contrarian angles. The OS provides the *judgment*.

Prompt Diff: Cause & Effect

What changed in the Innovations prompt?

+ "Every answer must contain: Consensus view, Strongest opposing view..."
+ "Every response should identify hidden leverage."

Result: The agent literally outputted these exact headers in the Startup Benchmark, forcing a smarter decision workflow.

The Core Finding

"We found that personality transfers. Intelligence largely doesn't."

Behavioral scaffolding successfully transferred tone, structure, and interaction style, but reasoning and knowledge remained tied to the underlying model.

Surprising Failures

  • Compression Degradation: Prompt compression preserved safety behavior but severely degraded conversational naturalness.
  • The "Founder Mode" Trade-off: Founder Mode improved startup analysis but actively hurt neutral information retrieval (the model over-optimized for monetization).
